SECTION 0 — HOW TO USE THIS DOCUMENT
This document is written for an AI coding assistant. Each numbered section below describes one problem found on the Sentinel AI website. Each problem entry follows a strict, machine-readable format:


* WHAT IT IS — a plain-English description of the problem.
* WHERE TO FIND IT — the exact HTML element, file, or route affected.
* WHY IT MATTERS — the user impact and business cost of leaving it unfixed.
* EXACT SOLUTION — the complete code, attribute changes, or architectural change needed. Copy-paste ready.
* ACCEPTANCE CRITERIA — a checklist of conditions that must pass before the issue is considered resolved.


The AI should work through this document sequentially. All code examples are complete and ready to implement unless a placeholder such as [YOUR_VALUE] appears — those must be filled in with project-specific data.


ISSUE 1 — ALL NAVIGATION LINKS ARE BROKEN
Severity: CRITICAL — Broken Functionality
Category: Functionality / Navigation


WHAT IT IS
The three top navigation links — "Products", "Resources", and "Contact" — all have href="#". When a user clicks any of them, the page simply scrolls to the very top. Nothing useful happens. The site is essentially unnavigable from the navbar.
WHERE TO FIND IT
In the main navigation HTML, look for the <nav> element. The links inside look like this:
<a href="#">Products</a>
<a href="#">Resources</a>
<a href="#">Contact</a>
WHY IT MATTERS
Navigation is the primary wayfinding system on a website. Non-functional navigation links create an immediate trust deficit. Users (and investors/judges at a hackathon) who click and go nowhere assume the product is incomplete. It also breaks accessibility since screen readers announce these as valid links.
EXACT SOLUTION
Option A — Anchor links to existing sections on the page (recommended for a single-page app):
Step 1: Add id attributes to the relevant sections in your HTML:
<section id="products"> ... </section>
<section id="resources"> ... </section>
<section id="contact"> ... </section>
Step 2: Update the navbar anchor hrefs to point to those IDs:
<a href="#products">Products</a>
<a href="#resources">Resources</a>
<a href="#contact">Contact</a>
Option B — If these are separate pages, use proper routes:
<a href="/products">Products</a>
<a href="/resources">Resources</a>
<a href="/contact">Contact</a>
Also add CSS smooth scrolling globally:
html { scroll-behavior: smooth; }
ACCEPTANCE CRITERIA
* Clicking "Products" in the navbar scrolls to or loads the Products section.
* Clicking "Resources" in the navbar scrolls to or loads a Resources section.
* Clicking "Contact" in the navbar scrolls to or loads a Contact section.
* No nav link points to a bare # anymore.


ISSUE 2 — HERO CTA BUTTONS DO NOTHING
Severity: CRITICAL — Broken Functionality
Category: Functionality / UX


WHAT IT IS
The three call-to-action buttons in the hero section — "Scan Message", "Check Call", and "Analyze URL" — do not trigger any visible action. They do not scroll the user to the scanner tool, open a modal, or change any state on the page. They appear entirely decorative, which defeats their entire purpose as the primary conversion buttons on the most-viewed section of the site.
WHERE TO FIND IT
These are the three <button> or <a> elements directly below the hero headline and subtitle, rendered in a horizontal row.
WHY IT MATTERS
These are the highest-visibility interactive elements on the entire site. A first-time visitor sees them first. If they click and nothing happens, they immediately lose trust and leave. Each button should be the fastest possible path to actually using the product.
EXACT SOLUTION
Each button should scroll smoothly to the corresponding section of the scanner:
// "Scan Message" button
document.querySelector("#btn-scan-message").addEventListener("click", () => {
  document.querySelector("#message-scanner").scrollIntoView({ behavior: "smooth" });
  // Also activate the "Text" tab if tabs exist
  activateTab("text");
});


// "Check Call" button
document.querySelector("#btn-check-call").addEventListener("click", () => {
  document.querySelector("#call-scanner").scrollIntoView({ behavior: "smooth" });
});


// "Analyze URL" button
document.querySelector("#btn-analyze-url").addEventListener("click", () => {
  document.querySelector("#url-scanner").scrollIntoView({ behavior: "smooth" });
});
Add the corresponding id attributes to the scanner sections: id="message-scanner", id="call-scanner", id="url-scanner".
If "Check Call" is not yet built, the button should instead show a "Coming Soon" modal or tooltip rather than doing nothing.
ACCEPTANCE CRITERIA
* Clicking "Scan Message" scrolls the viewport to the Message Scanner section.
* Clicking "Analyze URL" scrolls to the URL scanner section.
* "Check Call" either scrolls to a functional call scanner or shows a clear "Coming Soon" message.
* All scrolling uses smooth behavior, not instant jumps.


ISSUE 3 — MESSAGE SCANNER ANALYZE BUTTON IS NON-FUNCTIONAL
Severity: CRITICAL — Core Product Broken
Category: Core Product


WHAT IT IS
The "Analyze" button inside the Message Scanner section either does nothing, or submits the text without showing any result. There is no loading indicator, no result card, no error for empty input, and no visual feedback at all. This is the primary feature of the entire product and it does not work.
WHERE TO FIND IT
The Message Scanner section contains a textarea (for text input) and an "Analyze" button. The button's click handler is either missing, or it calls an API but the result display code is absent.
WHY IT MATTERS
This is literally the product. If the scanner does not work, there is no product to evaluate, use, or trust. Every visitor who tries it and sees nothing will leave and never return. This is the single most important thing to fix on the entire site.
EXACT SOLUTION — Complete Implementation
Step 1: HTML structure for the scanner with proper IDs:
<section id="message-scanner">
  <textarea id="scanner-input"
    placeholder="Paste a suspicious SMS, WhatsApp message, or email here...">
  </textarea>
  <button id="analyze-btn" onclick="analyzeMessage()">Analyze</button>
  <div id="scanner-result" style="display:none;"></div>
  <div id="scanner-error" style="display:none;"></div>
  <div id="scanner-loading" style="display:none;">Analyzing...</div>
</section>


Step 2: JavaScript function with full error handling:
async function analyzeMessage() {
  const input = document.getElementById("scanner-input").value.trim();
  const resultDiv = document.getElementById("scanner-result");
  const errorDiv = document.getElementById("scanner-error");
  const loadingDiv = document.getElementById("scanner-loading");


  // Input validation
  if (!input) {
    errorDiv.textContent = "Please enter a message to analyze.";
    errorDiv.style.display = "block";
    return;
  }


  // Reset UI
  errorDiv.style.display = "none";
  resultDiv.style.display = "none";
  loadingDiv.style.display = "block";


  try {
    const response = await fetch("/api/analyze", {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({ message: input })
    });
    if (!response.ok) throw new Error("Server error");
    const data = await response.json();


    // Show result card
    loadingDiv.style.display = "none";
    resultDiv.innerHTML = `
      <div class="result-card risk-${data.risk_level}">
        <h3>Risk Level: ${data.risk_label}</h3>
        <p>Confidence: ${data.confidence}%</p>
        <p>${data.explanation}</p>
        <ul>${data.indicators.map(i => `<li>${i}</li>`).join("")}</ul>
      </div>
    `;
    resultDiv.style.display = "block";


  } catch (err) {
    loadingDiv.style.display = "none";
    errorDiv.textContent = "Analysis failed. Please try again.";
    errorDiv.style.display = "block";
  }
}


Step 3: Add an "Try an Example" button that pre-fills a sample scam message:
<button onclick="fillExample()">Try an Example</button>
function fillExample() {
  document.getElementById("scanner-input").value =
    "URGENT: Your SBI account has been blocked. Click here to verify: http://sbi-secure.xyz/login";
}
ACCEPTANCE CRITERIA
* Clicking Analyze with an empty input shows a validation error message.
* Clicking Analyze with valid text shows a loading spinner while the request is in flight.
* A result card appears with at minimum: risk level, confidence score, and explanation.
* If the API call fails, a friendly error message appears (not a console error).
* The Try an Example button pre-fills a sample scam message.

// JavaScript: tab switching
function switchTab(tab) {
  document.getElementById("panel-text").hidden = (tab !== "text");
  document.getElementById("panel-image").hidden = (tab !== "image");
  document.getElementById("tab-text").setAttribute("aria-selected", tab === "text");
  document.getElementById("tab-image").setAttribute("aria-selected", tab === "image");
}


// Image preview on file select
document.getElementById("image-upload").addEventListener("change", (e) => {
  const file = e.target.files[0];
  if (!file) return;
  if (file.size > 5 * 1024 * 1024) {
    alert("File too large. Max 5MB.");
    return;
  }
  const reader = new FileReader();
  reader.onload = (ev) => {
    document.getElementById("image-preview").innerHTML =
      `<img src="${ev.target.result}" style="max-width:300px; border-radius:8px;">`;
  };
  reader.readAsDataURL(file);
});
ACCEPTANCE CRITERIA
* Clicking "Image" tab hides the textarea and shows the file upload input.
* Clicking "Text" tab shows the textarea and hides the file upload input.
* Uploading an image shows a thumbnail preview below the upload field.
* Files over 5MB are rejected with a user-facing error.
* The Analyze button works for both text and image inputs.

ISSUE 7 — STATISTICS HAVE NO SOURCE CITATIONS
Severity: MAJOR — Trust / Credibility
Category: Trust


WHAT IT IS
The stats section displays large, impactful numbers — "1,25,000 crore lost annually", "95,000+ daily scam attempts", "77% victims unaware" — with zero citations, sources, or methodology. A security product with unsourced statistics destroys credibility with technically literate users.
EXACT SOLUTION
Below each stat figure, add a small source citation. Example:
<div class="stat-card">
  <h3>1,25,000 crore</h3>
  <p>Lost to Scams Annually in India</p>
  <small class="source">
    Source: <a href="[URL]" target="_blank">
    Ministry of Home Affairs Cybercrime Report 2023</a>
  </small>
</div>
Suggested authoritative sources to research and cite:
* Ministry of Home Affairs Annual Report on Cybercrime — mha.gov.in
* NCRB (National Crime Records Bureau) — ncrb.gov.in
* RBI Annual Report on Payment Fraud — rbi.org.in
* CERT-IN (Indian Computer Emergency Response Team) — cert-in.org.in
Style for the source citation (small, muted, does not distract):
.source { font-size: 11px; color: #9E9E9E; display: block; margin-top: 4px; }
.source a { color: #1565C0; text-decoration: underline; }
ACCEPTANCE CRITERIA
* Every statistic has a visible source citation beneath it.
* At least two of the citations link to official government or research sources.
* The source text is styled subtly and does not compete visually with the stat.

ISSUE 12 — NO SMOOTH SCROLL BEHAVIOR
Severity: MODERATE — UX
Category: UX


WHAT IT IS
Anchor link navigation causes instant jumps to sections instead of smooth animated scrolling. This feels jarring and low-quality.
EXACT SOLUTION
Add one line to your global CSS file:
html {
  scroll-behavior: smooth;
}
For more control (e.g., custom easing), use JavaScript:
document.querySelectorAll("a[href^='#']").forEach(anchor => {
  anchor.addEventListener("click", function(e) {
    e.preventDefault();

  });
});

ISSUE 13 — NO ACTIVE STATE ON NAV LINKS WHILE SCROLLING
Severity: MODERATE — UX
Category: UX


WHAT IT IS
As the user scrolls down the page, the navbar gives no indication of which section is currently in view. Active navigation highlighting is standard on professional single-page sites and greatly improves orientation.
EXACT SOLUTION
Use the Intersection Observer API:
const sections = document.querySelectorAll("section[id]");
const navLinks = document.querySelectorAll("nav a[href^='#']");


const observer = new IntersectionObserver((entries) => {
  entries.forEach(entry => {
    if (entry.isIntersecting) {
      navLinks.forEach(link => link.classList.remove("active"));
      const activeLink = document.querySelector(
        `nav a[href="#${entry.target.id}"]`
      );
      if (activeLink) activeLink.classList.add("active");
    }
  });
}, { threshold: 0.4 });


sections.forEach(s => observer.observe(s));
Add CSS for the active state:
nav a.active {
  color: #1565C0;
  border-bottom: 2px solid #1565C0;
  font-weight: 600;
}
ACCEPTANCE CRITERIA
* As the user scrolls to each section, the corresponding nav link is highlighted.
* Only one nav link is highlighted at a time.

ISSUE 16 — NO KEYBOARD / ARIA ACCESSIBILITY

WHAT IT IS
